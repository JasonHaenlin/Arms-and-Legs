
1. Déplacer le fichier assembler.jar dans la meme repertoire que le fichier source 


2. Lancer la commande :

		java -jar assembler.jar -p "nom_du_fichier_source"

		
3. L'assemleur genere alors trois fichiers :
		- rom_nom_du_fichier_source : le fichier d'initialisation de la ROM
		- ram_nom_du_fichier_source  : le fichier d'initialisation de la RAM
		- map_nom_du_fichier_source  : le fichier qui sera utiliser pour afficher les valeur des variables a la fin de l'execution
		
		
4. Ouvrir le circuit "ordinateur.circ"



5. Initialiser la RAM et la ROM avec les fichiers générés



6. Lancer la simulation avec "ticks Enabled" 
   (IMPORTANT : Et juste avant de lancer la simulation veuillez reverifier que la RAM n'a pas été remise a 0 par une quelconque manipulation  : IMPORTANT)



7. Une fois la simulation démarrée, attendre que la lampe "program Ending" s'allume. L'allumage de la lampe signale que l'execution du programme est bien terminée



8. Arreter la simulation, et enregister le contenu de la RAM dans un autre fichier ( Exemple : "ram_final" ) pour afficher les valeurs des variables apres execution, avec le dumper.



9. Pour utiliser le dumper, lancer la commande :

		java -jar dumper.jar -r "ram_final" -m "map_nom_du_fichier_source"

		
10. Cette commande vous affiche le contenu de chaque variable sous le format : "nom_variable"="valeur_final"


11.Pour reexecuter un autre programme sans fermer logisim, appuyer sur le bouton "Main Reset", ce bouton reinitialise tous les circuits (sauf la ROM), puis reprendre a l'etape 1.
